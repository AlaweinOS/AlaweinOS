# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security issue in Optilibria, please report it by emailing:

**Email**: meshal@berkeley.edu

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Timeline
- Initial response: Within 48 hours
- Status update: Within 7 days
- Fix timeline: Depends on severity (critical issues prioritized)

### Security Best Practices

When using Optilibria:
1. **Input Validation**: Always validate problem inputs before optimization
2. **Resource Limits**: Set appropriate time/iteration limits for optimization runs
3. **Dependencies**: Keep all dependencies up to date
4. **Sensitive Data**: Do not include sensitive data in optimization problem definitions that may be logged

## Disclosure Policy

- Vulnerabilities will be disclosed after a fix is available
- Credit will be given to reporters (unless anonymity is requested)
- CVE IDs will be requested for significant vulnerabilities
